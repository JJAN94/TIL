# SQL에서 알면 좋은 내용

- 알아두면 좋은 내용 정리



## cmd에서 mysql 접속하기

```
mysql -u root -p
```

그 다음 패스워드를 입력하면 MySQL에 접속이 된다.



## USE

쿼리창에서 사용할 데이터베이스 이름을 치면 다음의 데이터베이스를 이용한다.

```mysql
USE 데이터베이스_이름;
```



## 대소문자

SQL에선 일반적으로 대소문자를 구별하지 않는다. 다만 가독성을 위해서 예약어는 대문자로, 사용자 정의어 등은 대소문자를 상황에 맞게 쓰자.

## 현재 데이터 베이스에 있는 테이블 정보 조회

```mysql
SHOW TABLE STATUS;
```

해당 데이터베이스에 있는 테이블에 대한 정보가 나온다. 테이블 이름, 행, 열 수 등등



단순히 테이블 이름만 볼 것이면 다음과 같이 검색하면 된다.

```mysql
SHOW TABLES;
```



## 테이블에 대한 정보 조회

employees 테이블에 대한 정보를 보자.

```mysql
DESCRIBE employees;

-- 또는

DESC employees;
```



## 별칭(alias)

열 이름을 별도의 원하는 이름으로 지정해서 조회할 수 있다. 열 이름 뒤 AS 별칭으로 하면된다.

```mysql
SELECT first_name AS 이름, gender 성별, hire_date '회사 입사일'
FROM employees;
```

AS는 생략해도 된다.

**단**, 별칭에 공백이 있다면 반드시 ' '로 씌어줘야 한다.



## 데이터타입에 대한 짤막 팁 - mysql

CHAR, VARCHAR - 영문자 1byte
NCHAR, NVARCHAR - 유니코드 2byte

--> 영문자 쓸 경우 CHAR, VARCHAR, 한글 쓸 경우 NCHAR, NVARCHAR사용

mysql 8.0인 경우 CHAR, VARCHAR 모두 'utf-8' 코드를 사용.
utf-8인 경우 영문/숫자/기호는 1byte, 한글/중어/일어 등은 3byte할당되므로 NCHAR, NVARCHAR를 쓸 필요가 없다.

그러므로 예를 들어 CHAR(10)이라 하면 영어든, 한글이든 10자까지 입력할 수 있다.



## 데이터베이스 개체 이름 규칙

DB(=schemas) 개체의 이름을 식별자(identifier)라고 한다.

여기서 DB 개체란?

- 데이터베이스, 테이블, 인덱스, 열, 인덱스, 뷰, 트리거, 스토어드 프로시저 등과 같은 개체들을 의미한다.

개체들의 이름규칙?

- 영문자 대소문자, 0~9, $, _ 사용가능(대소문자 구분없이 일단 소문자로 생성된다.)
- 개체 이름은 최대 64자 limit
- 예약어 사용 불가(select, from... 등)
- 개체 이름은 원칙상 공백 사용 불가. 만약 꼭 써야하면 \`My SQL\` 처럼 \`를 넣어줘야 한다.
- 개체이름은 딱 봐도 '아~ 뭐겠구나' 하는 이름으로 지어주자. 성의없이 abc같이 지어주면 뭔지 모른다.
- 리눅스는 db이름과 테이블 이름 모두 소문자로 써야한다. windows, linux호환을 생각하면 다 소문자로 써주자.
